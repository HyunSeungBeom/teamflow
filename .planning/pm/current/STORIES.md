# Sprint 2 — 유저 스토리

---

## US-S2-001: 프로젝트 워크스페이스 진입 (REQ-WS-LAYOUT-001, REQ-CARRY-002)

**As a** 로그인된 팀 멤버
**I want to** 프로젝트 허브에서 프로젝트 카드를 클릭하면 해당 프로젝트의 워크스페이스로 진입하고 싶다
**So that** 프로젝트의 이슈, 문서, 채팅 등 협업 기능을 사용할 수 있다

### 인수 기준
- [ ] 프로젝트 카드 클릭 시 `/workspace/:wsNo/project/:projectNo` 라우트로 이동
- [ ] 사이드바에 프로젝트명 + 탭 목록(Board, Docs, Chat) 표시
- [ ] Board 탭이 기본 선택 상태
- [ ] Docs, Chat 탭은 "Coming Soon" 비활성 상태로 표시
- [ ] 사이드바 상단에 허브 복귀 버튼 (← 또는 로고 클릭)

## US-S2-002: 프로젝트 생성 (REQ-CARRY-001)

**As a** 워크스페이스 멤버
**I want to** 프로젝트 허브에서 새 프로젝트를 만들고 싶다
**So that** 팀의 새로운 작업 공간을 생성할 수 있다

### 인수 기준
- [ ] "+ 새 프로젝트" 버튼 클릭 → 프로젝트 생성 모달 표시
- [ ] 프로젝트 이름 입력 (필수, 2~100자)
- [ ] "생성" 버튼 → POST /api/workspaces/:wsNo/projects → 성공 시 모달 닫기 + 목록 갱신
- [ ] 생성된 프로젝트가 즉시 카드 목록에 나타남

## US-S2-003: 이슈 생성 (REQ-ISSUE-001)

**As a** 프로젝트 멤버
**I want to** 칸반 보드에서 새 이슈를 만들고 싶다
**So that** 팀의 할 일을 추적하고 관리할 수 있다

### 인수 기준
- [ ] "+ 이슈 만들기" 버튼 → 이슈 생성 모달/폼 표시
- [ ] 필수: 제목 (2~200자)
- [ ] 선택: 설명, 상태(기본 BACKLOG), 담당자, 우선순위(LOW/MEDIUM/HIGH/CRITICAL), 기한
- [ ] 이슈 키 자동 생성: `{project.key}-{ticket_counter}` (예: TF-1, TF-2)
- [ ] 생성 후 칸반 보드에 즉시 반영

## US-S2-004: 칸반 보드 조회 (REQ-ISSUE-002)

**As a** 프로젝트 멤버
**I want to** 이슈들을 상태별로 분류된 칸반 보드에서 보고 싶다
**So that** 팀의 작업 현황을 한눈에 파악할 수 있다

### 인수 기준
- [ ] 4개 컬럼: BACKLOG, TODO, IN_PROGRESS, DONE
- [ ] 각 컬럼에 해당 상태의 이슈 카드 표시
- [ ] 카드에 이슈 키, 제목, 우선순위 아이콘, 담당자 아바타, 기한 표시
- [ ] 컬럼 헤더에 이슈 개수 표시
- [ ] 이슈가 없는 컬럼은 빈 상태 표시

## US-S2-005: 드래그 앤 드롭으로 상태 변경 (REQ-ISSUE-003)

**As a** 프로젝트 멤버
**I want to** 이슈 카드를 드래그해서 다른 컬럼으로 옮기고 싶다
**So that** 빠르게 이슈 상태를 변경할 수 있다

### 인수 기준
- [ ] 카드를 드래그하여 다른 컬럼에 드롭 → 상태 자동 변경 (PATCH API)
- [ ] 같은 컬럼 내에서 순서 변경 가능
- [ ] 드래그 중 시각적 피드백 (카드 투명도, 드롭 영역 하이라이트)
- [ ] 서버 오류 시 원래 위치로 롤백 (낙관적 업데이트 + 실패 복구)

## US-S2-006: 이슈 상세 보기/편집 (REQ-ISSUE-004, 005)

**As a** 프로젝트 멤버
**I want to** 이슈 카드를 클릭하면 상세 정보를 보고 편집하고 싶다
**So that** 이슈의 세부사항을 확인하고 업데이트할 수 있다

### 인수 기준
- [ ] 카드 클릭 → 사이드 패널로 상세 정보 표시
- [ ] 인라인 편집: 제목, 설명 직접 수정
- [ ] 드롭다운: 상태, 담당자, 우선순위 변경
- [ ] 날짜 선택: 기한 변경
- [ ] 저장 → PATCH API → 보드에 즉시 반영
- [ ] ESC 또는 배경 클릭 → 패널 닫기

## US-S2-007: 이슈 삭제 (REQ-ISSUE-006)

**As a** 프로젝트 멤버
**I want to** 불필요한 이슈를 삭제하고 싶다
**So that** 보드를 깔끔하게 유지할 수 있다

### 인수 기준
- [ ] 이슈 상세 패널에서 "삭제" 버튼
- [ ] 확인 다이얼로그: "이 이슈를 삭제하시겠습니까?"
- [ ] 확인 → DELETE API → 보드에서 카드 제거
- [ ] 취소 → 다이얼로그 닫기
