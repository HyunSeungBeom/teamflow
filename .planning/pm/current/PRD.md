## 1. 개요

### 1.1 목적
프로젝트 허브에서 프로젝트를 선택하면 진입하는 **프로젝트 워크스페이스**를 구축하고, 그 안에서 **이슈 관리(칸반 보드)**를 할 수 있도록 한다. TeamFlow의 첫 번째 핵심 협업 기능.

### 1.2 스프린트 목표
> 프로젝트 카드 클릭 → 워크스페이스 진입 → 칸반 보드에서 이슈 생성/관리/상태 변경 전체 플로우 완성

### 1.3 대상 사용자
- 소규모 팀(2~15명)의 프로젝트 매니저/리더
- 개발자, 디자이너 등 이슈를 생성하고 상태를 추적하는 팀 멤버

### 1.4 성공 기준
- 프로젝트 허브에서 프로젝트 카드 클릭 → 워크스페이스 진입
- 칸반 보드에서 이슈 생성/조회/수정/삭제 가능
- 드래그 앤 드롭으로 이슈 상태 변경 (Backlog → To Do → In Progress → Done)
- 이슈 상세 보기/편집 (제목, 설명, 담당자, 우선순위, 기한)

### 1.5 비목표 (Non-Goals)
- 문서 관리(Notion 기능) — Sprint 3
- 채팅(Slack 기능) — Sprint 3~4
- 이슈 댓글(ISS-05), 라벨/태그(ISS-06), 필터(ISS-07) — Sprint 4+ (P1)
- 리스트 뷰(ISS-08), 타임라인(ISS-09), 스프린트 관리(ISS-10) — Sprint 7+ (P2)
- 이슈 알림/멘션
- 이메일 기반 초대 (Sprint 1 이월, Sprint 3으로 재이관)

## 2. 요구사항

### 2.1 워크스페이스 레이아웃

| ID | 기능 | 우선순위 | 설명 |
|----|------|----------|------|
| REQ-WS-LAYOUT-001 | 워크스페이스 진입 | P0 | 프로젝트 허브 카드 클릭 → `/workspace/:wsNo/project/:projectNo` |
| REQ-WS-LAYOUT-002 | 사이드바 | P0 | 프로젝트명 + 탭 네비게이션 (Board 활성, Docs/Chat 비활성 표시) |
| REQ-WS-LAYOUT-003 | 메인 콘텐츠 영역 | P0 | 선택된 탭 콘텐츠 렌더링 (Sprint 2는 Board 탭만) |
| REQ-WS-LAYOUT-004 | 허브 복귀 | P0 | 사이드바 로고 또는 뒤로가기 → 프로젝트 허브로 이동 |

### 2.2 이슈 관리

| ID | 기능 | 우선순위 | 설명 |
|----|------|----------|------|
| REQ-ISSUE-001 | 이슈 생성 | P0 | 제목(필수), 설명, 상태, 담당자, 우선순위, 기한. 이슈 키 = `{project.key}-{ticket_counter}` 자동 생성 |
| REQ-ISSUE-002 | 칸반 보드 | P0 | 4컬럼: BACKLOG, TODO, IN_PROGRESS, DONE. 각 컬럼에 이슈 카드 표시 |
| REQ-ISSUE-003 | 드래그 앤 드롭 | P0 | 카드를 컬럼 간 드래그하여 상태 변경. 같은 컬럼 내 순서 변경도 지원 |
| REQ-ISSUE-004 | 이슈 상세 | P0 | 카드 클릭 → 사이드 패널로 상세 정보 표시. 인라인 편집 |
| REQ-ISSUE-005 | 이슈 수정 | P0 | 제목, 설명, 담당자, 우선순위, 기한, 상태 변경 |
| REQ-ISSUE-006 | 이슈 삭제 | P0 | 확인 다이얼로그 후 삭제 |

### 2.3 Sprint 1 이월 항목

| ID | 기능 | 우선순위 | 설명 |
|----|------|----------|------|
| REQ-CARRY-001 | 프로젝트 생성 모달 | P0 | 프로젝트 허브에서 프로젝트 생성 (이름 입력) |
| REQ-CARRY-002 | 프로젝트 진입 라우팅 | P0 | 카드 클릭 → 워크스페이스 라우팅 연결 |
